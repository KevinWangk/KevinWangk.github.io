业务应用场景：解决三高需求

  • 高并发读写
  • 高效率存储与访问
  • 数据库高拓展性、高可用性

使用特点：

  • 数据量大
  • 写入操作频繁
  • 价值较低的数据，对事务要求不高

简介

数据结构松散，使用类似于json的格式bson。

数据库–>Collection–>Document–> ( field:value ) 相当于MySQL中的数据库–>表(table)–>行(row)–>(column)

MongoDB中的记录是一个文档，有字段和值对（field:value）构成
